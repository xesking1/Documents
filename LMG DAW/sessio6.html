elementos de bloque

address

representa la información de contacto para el elemento article o para el elemento body. será uno u otro en función de lo que tenga más cerca

ejemplo:

<address>Puede contactar con el centro a través del link < a href="https://paucasesnovescifp.cat/"> https://paucasesnovescifp.cat/</a ></address>

article

Es un contenedor que agrupa información concebida como una unidad independiente.

Podríamos hacer el símil con un artículo de un periódico, una entrega de un blog, etc.

aside

agrupa contenidos que están relacionados con la información principal de la página.

Aunque no tiene una posición preestablecida en muchas ocasiones se coloca en una posición lateral.

En su interior se pueden encontrar enlaces relacionados, publicidad relacionada o información complementaria a la información principal

dialog

mostrar una caja de diálogo

si aparece el atributo open, el contenido de la caja de dialogo se mostrara

ejemplo:

la compra se ha realizado correctamente!!

div

Permite crear divisiones y agrupar contenido. se suele utilizar para aplicar estilos a los elementos agrupados

dl, dt y dd

Permiten definir una lista de descripciones

dl
Marca la lista de definición
dt
Cada uno de los terminos definidos
dd
La propia definición del término

footer

Contenedor para la información de pie de página

Normalmente contiene información sobre quién ha realizado la página web, copyright, enlaces de interés, soporte, etc.

h1-h6

Representan los encabezados o títulos

Cada elemento tiene una representación distinta en el navegador de mayorr importancia a menor importancia

Los agentes de usuario utilizan los encabezados para construir las tablas de contenido del sitio web, lo que mejora el posicionamiento SEO. Por ello es importante usarlos correctamente

Solo puede haber un h1 por página y no puede aparecer una etiqueta si de un nivel si el nivel anterior no se ha utilizado.
